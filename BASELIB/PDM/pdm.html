Routinen zum Arbeiten mit Pull-Down-FensternRoutines for Working with Pull-Down Windows

Die hier vorgestellten Routinen sind solche für das Arbeiten mit Pull-Dowm-Menues. Diese sind als Test zu betrachten, da ich in der Praxis kaum damit gearbeitet habe. Die Routinen benötigen die Library PDM.LIB, die mittels MACLIB PDM.LIB eingefügt wird. Einige Hinweise zu der Struktur findet sich hier. Zu beachten ist, dass diese Routinen speziell für den JOYCE konzipiert sind.The routines presented here are those for working with pull-down menus. These should be considered experimental, as I have hardly worked with them in practice. The routines require the library PDM.LIB, which is included via MACLIB PDM.LIB. Some notes on the structure can be found here. Note that these routines are specifically designed for the JOYCE.

PDMCUL.MAC
PDMCUL B: Eingabe eines Elementes aus einem FensterGet logic input from a window element
  I: Registerpaar DE hält UPB. Registerpaar HL hält alle Elemente. Register B hält Kontrollbyte für Modul PDMCUI, Register C hält Kontrollbyte für Modul PDMTXO, Akku hält Nummer des Elements, mit dem begonnen werden soll. Zusätzlich definiert Bit 5 im Register B, ob das Fenster neu aufgebaut werden soll (Bit nicht gesetzt). Dies ist sinnvoll bei einem Fenster als Hauptmenue, das bis zum Ende eines Programms bestehen bleibtRegister pair DE holds UPB. Register pair HL holds all elements. Register B holds control byte for module PDMCUI, Register C holds control byte for module PDMTXO, accumulator holds the number of the element to start with. Additionally, bit 5 in register B defines whether the window should be rebuilt (bit not set). This is useful for a window used as a main menu that persists until the end of a program
  O: Register C hält die Nummer des gewählten Elementes, die Zeroflag zeigt RETURN oder ENTER anRegister C holds the number of the selected element, the zero flag indicates RETURN or ENTER

PDMBIC
PDMXBS		   I: Registerpaar HL hält verfügbare Speicheradresse im Common Bereich (größer hex C000)Register pair HL holds the available memory address in the Common area (greater than hex C000)
  O: Die Carryflag zeigt an, dass Common Bereich nicht ausreicht. Registerpaar HL und Feld PDMTOP halten neue verfügbare höchste Adresse. PDMTKX B: Tastenwert ermittelnGet key value

Diese Routine sollte dem Modul PDMGTK vorgezogen werden. Zuvor muss PDMXOS aufgerufen werden.The accumulator holds the corresponding value, the carry flag indicates an invalid key number.
This routine should be preferred over module PDMGTK. PDMXOS must be called beforehand.
