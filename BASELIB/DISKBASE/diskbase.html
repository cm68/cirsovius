Routinen für DiskettenoperationenRoutines for Disk Operations

Die hier vorgestellten Routinen sind solche für die Verabeitung von grundlegenden Diskoperationen. Dies umfasst Parse-Routinen sowie Schnittstellen direkt zum BDOS. Die I/O Operationen sind Rekord-basiert:The routines presented here are those for processing fundamental disk operations. This includes parse routines as well as interfaces directly to the BDOS. The I/O operations are record-based:
Modul-EntryModule Entry B: Beschreibung, I: Eingabe-, O: AusgabeparameterDescription, I: Input, O: Output parameters

@BDOS IORES B: Ausführen eines BDOS Aufrufes für DiskettenoperationenExecute a BDOS call for disk operations
  I: Registerpaar DE zeigt auf einen FCB, der Akku hält die auszuführende BDOS FunktionRegister pair DE points to an FCB, the accumulator holds the BDOS function number to execute
  O: Das Ergebnisfeld IORES hält den Inhalt des Akkus, IORES+1 das Register H und IORES+2 die BDOS FunktionThe result field IORES holds the content of the accumulator, IORES+1 holds register H, and IORES+2 holds the BDOS function
FILCHK.MAC
FILCHK @FILCK B: Testen der CP/M Parameter nach Aufruf eines ProgrammesTest legal CP/M file parameters after program invocation
  I: Bei Aufruf von FILCHK muss der Standard File Control Block (FCB) legale Dateiparameter beinhalten. Bei Aufruf von @FILCK weist Registerpaar DE auf ein FCB. Registerpaar HL zeigt auf eine Liste mit zwei Adressen, die im Fehlerfall geladen werden.When calling FILCHK, the standard File Control Block (FCB) must contain legal file parameters. When calling @FILCK, register pair DE points to an FCB. Register pair HL points to a list with two addresses that are loaded in case of error.
  O: Die Carryflag ist nicht gesetzt, wenn in der FCB ein legaler Dateiname vorhanden ist. Dann hält das Registerpaar DE die Adresse der FCB. Die Carryflag ist gesetzt, wenn ein illegaler oder leerer Dateiname gefunden wurde, dabei hält Registerpaar DE die erste Adresse aus der Liste, wenn kein Dateiname gefunden wurde, und die zweite Adresse, wenn der Dateiname fehlerhaft ist.The carry flag is reset if the FCB contains a legal file name. In that case, register pair DE holds the address of the FCB. The carry flag is set if an illegal or empty file name was found; register pair DE then holds the first address from the list if no file name was specified, or the second address if the file name is illegal.

GETDU B: Umwandeln einer Zeichenkette in Laufwerk mit optionaler Angabe des Nutzerbereiches. Diese Routine sollte vor einer der PARSE Routinen aufgerufen werden.Convert a string into a drive with optional user area specification. This routine should be called before one of the PARSE routines.
  I: Registerpaar HL zeigt auf die Zeichenkette, die mit einem Doppelpunkt abgeschlossen sein muss.Register pair HL points to the string, which must be terminated with a colon.
  O: Register B hält das Laufwerk und Register C hält den Nutzer. Register E zeigt an, ob ein Laufwerk und/oder Nutzer definiert wurde. Falls nicht, so stehen in den Registern die gerade eingestellten Werte.
  • Bit 0 : Gesetzt bei Eingabe eines Laufwerkes
  • Bit 1 : Gesetzt bei Eingabe eines Nutzers
Im Fehlerfall ist Carry gesetzt. Wenn kein Fehler vorliegt, dann zeigt das Registerpaar HL hinter das Zeichen „:" oder bleibt unverändert.Register B holds the drive and register C holds the user.   • Bit 0: Set if a drive was specified
  • Bit 1: Set if a user was specified
On error, the carry flag is set. If no error occurred, register pair HL points past the ":" character, or remains unchanged.

0:NUL:
1:CON:
2:AUX:
3:LST:
4+x:From extension (x=0, 1, etc.)

ERROR.MAC
@ERROR B: Überprüfen von Fehlern bei DiskettenoperationenCheck for errors during disk operations
  I: Gesetzte Carryflag zeigt Fehler an.Carry flag set indicates an error.
  O: Im Fehlerfall wird geprüft, ob Extended Mode eingestellt ist. Falls ja, dann wird getestet, ob im Feld ERRPC (siehe nächstes Modul) eine Adresse ungleich Null steht. Falls ja, verzweigt das Modul auf diese Adresse, andernfalls Rückkehr mit gesetzter Carryflag. Der Akku hält den Originalwert vom Register HOn error, it is checked whether extended mode is active. If so, it tests whether the field ERRPC (see next module) contains a non-zero address. If so, the module branches to that address; otherwise it returns with the carry flag set. The accumulator holds the original value from register H.
