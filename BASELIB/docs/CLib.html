Notes on the routines FOPEN, FCLOSE, FGETC, FPUTC, FGETS and FPUTS:Hinweise zu den Routinen FOPEN, FCLOSE, FGETC, FPUTC, FGETS und FPUTS:

These routines work with addresses of file control blocks or with device codes. Both are provided by the routine FOPEN. This routine converts a character string of the form DU:FN.EXT or DEV:. The meanings are:Diese Routinen arbeiten mit Adressen auf Dateikontrollblöcke oder mit Gerätekodes. Beides wird von der Routine FOPEN geliefert. Diese Routine wandelt eine Zeichenkette der Form DU:FN.EXT oder DEV: um. Es bedeuten:
DDrive A..PLaufwerk A..P
UUser number 0..15Benutzernummer 0..15
FNFile nameDateiname
EXTFile extensionDateiextension

In this case, a dynamic memory area is requested and returned. When FCLOSE is called, this memory area is released again. It comprises the space for the FCB (36 bytes), four records (512 bytes) and internal storage (6 bytes = 554 bytes).In diesem Fall wird ein dynamischer Speicherbereich angefordert und zurückgegeben. Bei Aufruf von FCLOSE wird dieser Speicherbereich wieder freigegeben. Er umfasst den Platz für den FCB (36 Bytes), vier Rekords (512 Bytes) und internen Speicher (6 Bytes = 554 Bytes).

For files, depending on the selected mode, the following action is performed:Bei Dateien wird, in Abhängigkeit vom gewählten Modus, folgende Aktivität ausgeführt:
r, orOpen a file for reading. If the file does not exist, an error is returned.Öffnen einer Datei zum Lesen. Ist die Datei nicht vorhanden, so wird ein Fehler zurückgegeben.
w, owOpen a file for writing. If a file exists, it is deleted first.Öffnen einer Datei zum Schreiben. Ist eine Datei vorhanden, so wird diese zuvor gelöscht.
aOpen a file for writing. If a file exists, the write pointer is positioned at the Ctrl-Z (end of file), otherwise a new file is created.Öffnen einer Datei zum Schreiben. Ist eine Datei vorhanden, so wird der Schreibzeiger auf das Ctrl-Z (End of file) positioniert, andernfalls wird eine neue Datei angelegt.
oaOpen a file for writing. If a file exists, the write pointer is positioned after the last record, otherwise a new file is created.Öffnen einer Datei zum Schreiben. Ist eine Datei vorhanden, so wird der Schreibzeiger hinter den letzten Rekord positioniert, andernfalls wird eine neue Datei angelegt.

For devices, the following code is generated:Für Geräte wird folgender Kode erzeugt:

NUL:w8

When calling the routines FGETC and FGETS, device code 1 can also be selected. In that case, the file or device defined in $STDIN is used (default CON:). When calling FPUTC and FPUTS, code 2 is allowed, using the definition in $STDOUT (default also CON:).Bei Aufruf der Routinen FGETC und FGETS kann als Gerätekode auch 1 gewählt werden. Dann wird die Datei oder das Gerät gewählt, das in $STDIN definiert ist (Voreinstellung CON:). Bei Aufruf von FPUTC und FPUTS ist Kode 2 erlaubt, es gilt die Definition in $STDOUT (Voreinstellung ebenfalls CON:).

bei der Ausgabe die Folge <LF> ausgegeben als <CR><LF>
