Darstellung der ZahlenNumber representation

 Der Zahlenbereich umfaßt 3.4 E38 bis -5.9 E-39 mit einer Genauigkeit von sieben Ziffern (Bei einer Mantissa mit 23 Bit sind das 2^-23 = 1.2 E-7).The number range covers 3.4 E38 to -5.9 E-39 with a precision of seven digits (With a 23 bit mantissa that is 2^-23 = 1.2 E-7).

Die Zahlen werden kodiert alsThe numbers are encoded as

Byte 1Byte 1Normalisierte Mantisse, MSB (mit Vorzeichen)Normalized mantissa, MSB (with sign)
Byte 2Byte 2MantisseMantissa
Byte 3Byte 3Mantisse, LSBMantissa, LSB
Byte 4Byte 4Exponent in binärer 2'er KomplementdarstellungExponent in binary two's complement representation

Die Mantisse erklärt sich wie folgt:The mantissa is explained as follows:
Bei einer Länge von drei Bytes ergeben sich 3*8=24 Bits. Das MSB wird als Vorzeichen benutzt. Das nächste Bit hat eine Wertigkeit von 1/2, das nächste Bit 1/4, dann 1/8 usw. Hieraus ersieht man, daß der Wert der Mantisse maximal 1/2+1/4+1/8+..., also immer < 1 ist. Der Ausdruck "Normalisierung" bedeutet hierbei, daß immer das dem MSB folgende Bit gesetzt ist, die Mantisse also immer >= 0,5 ist. Weil das MSB als Vorzeichen benutzt wird, muß der ermittelte Wert noch mit zwei multipliziert werden und danach muß nur noch der so ermittelte Wert der mit dem Exponenten multipliziert werden, um den realen Zahlenwert zu erhalten.With a length of three bytes, there are 3*8=24 bits. The MSB is used as the sign. The next bit has a weight of 1/2, the next bit 1/4, then 1/8 etc. From this one can see that the value of the mantissa is at most 1/2+1/4+1/8+..., thus always < 1. The term "normalization" means that the bit following the MSB is always set, so the mantissa is always >= 0.5. Because the MSB is used as the sign, the determined value must still be multiplied by two, and then this value only needs to be multiplied by the exponent to obtain the actual numerical value.

Die Zahl ist 0, wenn das dem MSB folgende Bit Null ist.The number is 0 when the bit following the MSB is zero.

Beispiel 1:Example 1:Zahl=1. Sie ist darstellbar durch 0,5*2^1=(0,5*2)*2^0, damit ergibt sich:Number=1. It can be represented by 0.5*2^1=(0.5*2)*2^0, thus we get:
40 80 00 00
Beispiel 2:Example 2:Zahl=-2,5. Sie ist darstellbar durch -(0,5+0,125)*2^2=-((0,5+0,125)*2)*2^1, damit ergibt sich für die Darstellung:Number=-2.5. It can be represented by -(0.5+0.125)*2^2=-((0.5+0.125)*2)*2^1, thus the representation is:
D0 00 00 01

FehlerbehandlungError handling

 Bei Bereichsüberschreitung der Zahlen, Wurzel aus einer negativen Zahl, Division durch Null und Zahlenüber- oder -unterlauf wird die interne Fehlerflag ‚FLPERR' auf den Wert TRUE (dezimal 255, hex FF) gesetzt. Außerdem wird bei einem Fehler das Carryflag der CPU gesetzt. Das Prüfen dieser Flag obliegt dem Programmierer. Im Speicherplatz ‚FLPERR+1' findet sich ein weiterer Code, der genauere Auskunft gibt:When numbers exceed the range, square root of a negative number, division by zero, and number overflow or underflow occur, the internal error flag ‚FLPERR' is set to the value TRUE (decimal 255, hex FF). Additionally, on error the CPU carry flag is set. Checking this flag is the programmer's responsibility. In the memory location ‚FLPERR+1' there is an additional code that provides more detailed information:

CodeBedeutungMeaning
0Overflow
2Division durch NullDivision by zero
4Exponent erwartetExponent expected
6Ziffer erwartetDigit expected
8Zahl zu großNumber too large
10Aufruf FehlerCall error
