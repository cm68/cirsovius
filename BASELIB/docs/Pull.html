Das Kontrollbyte (UPB+4) ist definiert als:The control byte (UPB+4) is defined as:
BitFunktionFunction
0FensterelementWindow element
  • 0 : Einfacher Strich0 : Single line
  • 1 : Doppelter Strich1 : Double line
1ZwischenraumSpacing
  • 0 : Kein Zwischenraum0 : No spacing
  • 1 : Zwischenraum1 : Spacing
2FensterrichtungWindow direction
  • 0 : HorizontalHorizontal
  • 1 : VertikalVertical
3FensterfarbeWindow color
  • 0 : Normale Darstellung0 : Normal display
  • 1 : Invertierte Darstellung1 : Inverted display
4Cursor rettenSave cursor
  • 0 : Nicht retten0 : Do not save
  • 1 : Retten (in UPB+S,UPB+6)1 : Save (in UPB+5,UPB+6)
5Fenster rettenSave window
  • 0 : Nicht retten0 : Do not save
  • 1 : Retten (in Adresse aus UPB+7,8)1 : Save (at address from UPB+7,8)
6Art des ZwischenraumsType of spacing
  • 0 : Strich als Zwischenraum0 : Line as spacing
  • 1 : Leerzeichen als Zwischenraum1 : Space as spacing
7Fenster zurückladenRestore window
  • 0 : Nicht zurückladen0 : Do not restore
  • 1 : Zurückladen (von Adresse aus UPB+7,8)1 : Restore (from address at UPB+7,8)

SpeicherbedarfMemory Requirements

 Der PCW speichert jedes Zeichen ab als eine 8x8 Bit Matrix, also in 8 Bytes.The PCW stores each character as an 8x8 bit matrix, i.e. in 8 bytes. Dies ist zu berücksichtigen bei der Bereitstellung von Speicher zum Retten von Fenstern.This must be taken into account when allocating memory for saving windows. Außerdem ist der unterschiedliche Speicherbedarf für horizontale und vertikale Richtung zu beachten.Additionally, the different memory requirements for horizontal and vertical directions must be considered.
