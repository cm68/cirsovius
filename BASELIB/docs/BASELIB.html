Meine ersten Programmierversuche am JOYCE waren mit dem CP/M 2.2 Assembler ASM, weil ich damals nur von ihm ein Handbuch vorliegen hatte. Völlig unbefriedigend war die Tatsache, dass beim ASM eine einzige Datei als Quelle vorliegen musste - bei längeren Programmen eine Strapaze! Als ich dann Unterlagen über den RMAC bekam, wurde dies anders, weil es der RMAC erlaubte, verschiebbare .REL (relocatable, relozierbar) Dateien zu erzeugen, die später dann mit einem Linker zu einer Datei zusammengefügt werden konnten.My first programming attempts on the JOYCE were with the CP/M 2.2 assembler ASM, because at that time it was the only one for which I had a manual available. Completely unsatisfying was the fact that ASM required a single file as source - a burden for longer programs! When I then got documentation for the RMAC, things changed, because RMAC allowed the creation of relocatable .REL files that could later be linked together into a single file using a linker.
Der Vorteil ist also hier, dass vorab Programmteile (Module) assembliert werden können. Diese können dann mit einem Hilfsprogramm (Librarian) in einer Bibliothek verwaltet werden. Nach dem Assemblieren des Hauptprogrammes (das auch wieder aus einzelnen Moduln bestehen kann), wird dann der Linker angewiesen, aus der Bibliothek benötigte Module dazuzuladen.The advantage here is that program parts (modules) can be assembled in advance. These can then be managed in a library using a utility program (librarian). After assembling the main program (which can also consist of individual modules), the linker is instructed to load required modules from the library.
Ich habe damals folgendes Entwicklungspaket verwendet:At that time I used the following development package:
  • Assembler: Zunächst den RMAC von Digital Research, der zum Lieferumfang von CP/M PLUS gehörte. Danach den M80 von Microsoft. Zusätzlich zum RMAC, der lediglich die 8080-Mnemonics kannte, konnte nun auch in Z80-Mnemonic programmiert werden. Später kam dann der SLR-Assembler, der sehr schnell war, allerdings nur Z80-Mnemonics erkannte.Initially the RMAC from Digital Research, which was included with CP/M PLUS. Then the M80 from Microsoft. In addition to RMAC, which only knew 8080 mnemonics, it was now possible to program in Z80 mnemonics. Later came the SLR assembler, which was very fast but only recognized Z80 mnemonics.
    Allen Assemblern gemein ist, dass sie das Microsoft-REL-Format (hier englisch) unterstützen.All assemblers have in common that they support the Microsoft REL format (English version here).
  • Linker: Hier kam für mich von vornherein nur LINK von Digital Research zum Einsatz. Dieses Programm unterstützt die Generierung einer RSX-Datei und ansatzweise den Overlay-Mechanismus.For the linker, I used LINK from Digital Research from the very beginning. This program supports the generation of an RSX file and rudimentary overlay mechanism support.
  • Librarian: Auch hier habe ich ein Produkt von Digital Research verwendet, nämlich LIB. LIB unterstützt das indizierte REL-Format (IRL). Das bedeutet, dass Bibliotheken schneller durchsucht werden. Hier eine Beschreibung des DR-IRL-Format (hier englisch).Here too I used a Digital Research product, namely LIB. LIB supports the indexed REL format (IRL). This means that libraries are searched more quickly. Here is a description of the DR IRL format (English version here).
  • Debugger: Es gibt einige komfortablere Debugger, aber auch hier habe ich ein Produkt von Digital Research berwendet, zuerst SID, später dann ZSID. ZSID hat die gleiche Funktionalität wie SID, jedoch mit Unterstützung der Z80-Syntax. Allerdings gab es anfangs Probleme mit dem Breakpoint - hier gab es Konflikte mit dem JOYCE-Interruptvektor.There are some more comfortable debuggers, but here too I used a Digital Research product, first SID, later ZSID. ZSID has the same functionality as SID, but with Z80 syntax support. However, there were initially problems with the breakpoint - there were conflicts with the JOYCE interrupt vector.

Es gibt zwei weitere erwähnenswerte Bibliotheken, mit denen ich gearbeitet habe:There are two other noteworthy libraries that I have worked with:
  • SYSLIB ist eine Sammlung von Routinen hauptsächlich für das ZCPR-Umfeld. (Hätte ich SYSLIB schon gekannt, als ich meine Library geschrieben hatte, hätte ich es mir erspart, das Rad neu zu erfinden). SYSLIB ist kompatibel zum Microsoft-REL-Format und besteht aus folgenden Modulen:is a collection of routines primarily for the ZCPR environment. (Had I already known SYSLIB when I wrote my library, I would have saved myself the trouble of reinventing the wheel). SYSLIB is compatible with the Microsoft REL format and consists of the following modules:
    • SYSLIB: allgemeine Routinengeneral routines
    • Z3LIB: ZCPR3 RoutinenZCPR3 routines
    • ZSLIB: ZSDOS Clock RoutinenZSDOS clock routines
    • VLIB: Videoroutinen für ZCPR3-Systemevideo routines for ZCPR3 systems
    • DSLIB: Clock Routinenclock routines
  • ZSMLIB ist eine Sammlung von C ähnlichen Routinen. Leider ist ZSMLIB nicht kompatibel zum Microsoft-REL-Format. Dafür gibt es einen eigenen Assembler, Linker und Librarian. Die Syntax des Assemblers ist allerdings anders als die des M80, do dass die Quellen ebenfalls nicht kompatibel sind.is a collection of C-like routines. Unfortunately, ZSMLIB is not compatible with the Microsoft REL format. Instead, it has its own assembler, linker, and librarian. The assembler syntax is however different from that of M80, so the sources are also not compatible.
