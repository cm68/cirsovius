Die hier vorgestellten Routinen sind solche für das allgemeine Arbeiten mit Dateien.The routines presented here are those for general file operations.

FILSIZ.MAC
FILSIZ B: Ermitteln der Größe einer DateiCompute file size
  I: Das Registerpaar DE hält Adresse der FCBRegister pair DE holds the address of the FCB
  O: Die Carryflag ist gesetzt falls Datei nicht vorhanden ist, sonst nicht gesetzt. Die Dateigröße in Rekords (128 Bytes) steht dann in FCB+33 bis FCB+35, beginnend mit niederwertigem Byte in FCB+33. Im Extended Error Mode im Fehlerfall Verzweigung möglich. Das Registerpaar HL hält das niederwertige und mittlere Byte, der Akkumulator hält das höchstwertige Byte. Die gesetzte Zeroflag zeigt an, dass das höherwertige Byte Null ist.The carry flag is set if the file is not found, otherwise it is reset. The file size in records (128 bytes) is then stored in FCB+33 through FCB+35, starting with the low byte in FCB+33. In Extended Error Mode, branching on error is possible. Register pair HL holds the low and middle byte, the accumulator holds the high byte. The zero flag being set indicates that the high byte is zero.

  I: Der Akku hält das Laufwerk : 0 -> A ... 15 -> PThe accumulator holds the drive: 0 -> A ... 15 -> P
  O: Die Carryflag ist gesetzt bei Laufwerksfehler und der Akku hält im Extended Error Mode den Fehlerkode, eine Verzweigung in die Fehlerroutine ist möglich. Im Normalfall hält der eingestellte DMA Puffer die Anzahl freier Rekords, DMA+0 hält das niederwertige und DMA+2 das höchstwertige ByteThe carry flag is set on drive error and the accumulator holds the error code in Extended Error Mode, branching to the error routine is possible. On normal return the current DMA buffer holds the number of free records, DMA+0 holds the low byte and DMA+2 the high byte

FILES.MAC
FILES B: Einlesen einer Directory in den SpeicherRead a directory into memory
  I: Register BC hält die maximale Anzahl zu lesender Einträge. Register HL weist auf den Start des Speicherbereiches. Register DE hält die Adresse des FCB, in dem Wildcards („?") benutzt werden können. Auch das Laufwerk wird hier definiert. (0 für aktuelles Laufwerk, 1 .. 16 für A .. P)Register BC holds the maximum number of entries to read. Register HL points to the start of the memory area. Register DE holds the address of the FCB, in which wildcards ("?") can be used. The drive is also defined here. (0 for current drive, 1 .. 16 for A .. P)
  O: Die Carryflag ist gesetzt bei nicht gefundener Datei. Das Register BC hält die tatsächliche Anzahl gefundener Einträge, Register HL weist auf den Start des Speicherbereiches und Register DE weist auf die erste freie Speicherstelle nach der Liste. Die nicht sortierte Liste ist gefüllt mit jeweils 12 Byte langen Einträgen. Das erste Byte ist hierbei die gefundene Benutzernummer und muss vor dem Gebrauch in einer FCB gegen ein Laufwerk getauscht werden.The carry flag is set if no file was found. Register BC holds the actual number of entries found, register HL points to the start of the memory area and register DE points to the first free memory location after the list. The unsorted list is filled with entries of 12 bytes each. The first byte is the user number found and must be replaced by a drive code before use in an FCB.

DIRGET.MAC
DIRGET B: Einlesen und Sortieren einer Directory in den SpeicherRead and sort a directory into memory
  I: Register C hält den Userbereich 0 .. 15. Register HL weist auf den Start des Speicherbereiches. Register DE hält die maximale Anzahl zu lesender Einträge. Der Akku hält die zu suchenden Dateiattribute in den Bits 7 (SYS), 6 (RO) und 5 (Ist Bit 5 gesetzt, werden Gruppen selektiert). Ist Bit 4 gesetzt, so werden die Dateiattribute so belassen, bei nicht gesetztem Bit 4 werden die Attribute gelöscht.Register C holds the user area 0 .. 15. Register HL points to the start of the memory area. Register DE holds the maximum number of entries to read. The accumulator holds the file attributes to search for in bits 7 (SYS), 6 (RO) and 5 (if bit 5 is set, groups are selected). If bit 4 is set, the file attributes are left as is; if bit 4 is not set, the attributes are stripped.
76.5<-Bits76.5<-Bits
00.0DIR01.1Alle SYSAll SYS
01.0SYS10.1Alle ROAll RO
10.0ROxx.1Alle DateienAll files
11.0SYS und ROSYS and RO
Der Standard File Control Block (FCB) in Adresse 005CH hält die Dateimaske der zu suchenden Dateien. Hier können Wildcards („?") benutzt werden. Auch das Laufwerk wird hier definiert. (0 für aktuelles Laufwerk, 1 .. 16 für A .. P)The standard File Control Block (FCB) at address 005CH holds the file mask for the files to search. Wildcards ("?") can be used here. The drive is also defined here. (0 for current drive, 1 .. 16 for A .. P)
  O: Die Carryflag ist gesetzt bei nicht existierendem Laufwerk und der Akku hält den Fehlerkode im Extended Error Mode, ebenfalls bei einem fehlerhaften Userbereich. Das Register HL hält die tatsächliche Anzahl gefundener Einträge und Register DE weist auf die erste freie Speicherstelle nach der Liste. Die sortierte Liste ist gefüllt mit jeweils 12 Byte langen Einträgen gefolgt von einer 16 Bit Adresse als Referenz auf das nächste Element. Das Feld SRFBUF muss die Adresse eines 128 Byte Puffers halten, die Voreinstellung ist der Standard DMA Puffer ab 0080HThe carry flag is set if the drive does not exist and the accumulator holds the error code in Extended Error Mode, likewise on an invalid user area. Register HL holds the actual number of entries found and register DE points to the first free memory location after the list. The sorted list is filled with entries of 12 bytes each followed by a 16-bit address as a reference to the next element. The field SRFBUF must hold the address of a 128-byte buffer; the default is the standard DMA buffer at 0080H
