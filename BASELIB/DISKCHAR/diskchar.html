Routinen für zeichenweise DiskettenoperationenRoutines for character-based disk operations

Die hier vorgestellten Routinen sind solche für zur Zeichen Ein-/Ausgabe bei Diskettenoperationen. Die BDOS-Schnittstelle ist in einem fünf Byte großen Parameterblock abgelegt (Siehe dazu die Module DSKGET und DSKPUT). Werden mehr als eine Datei bearbeitet, müssen diese Bytes jeweils umgepackt werden. Weil dies etwas umständlich ist, habe ich noch andere Routinen geschrieben.The routines presented here are for character-based input/output during disk operations. The BDOS interface is stored in a five-byte parameter block (see modules DSKGET and DSKPUT). If more than one file is being processed, these bytes must be swapped accordingly. Because this is somewhat cumbersome, I have written additional routines.

DSKGET.MAC
DSKGET   B: Lesen eines Bytes von DisketteRead a byte from disk input buffer, filling it with a new record if empty
  RDBUF   16 Bit DMA Puffer Adresse16 bit DMA buffer address
  RDBFP   8 Bit relative Pufferzeiger8 bit relative buffer pointer
  RDFCB   16 Bit FCB Adresse16 bit FCB address
  I: Die drei Felder müssen vor Aufruf gesetzt sein, falls nicht die Voreinstellungen verwendet werden sollenThe three fields must be set up prior to calling the module, unless the defaults are to be used
  O: Die Carryflag ist gesetzt falls alle Rekords gelesen wurden oder die Datei nicht gefunden wurde. Sonst hält der Akku das gelesene ByteCarry flag is set if all records have been read or the file was not found. Otherwise the accumulator holds the byte read
DSKPUT.MAC
DSKPUT   B: Schreiben eines Bytes auf DisketteWrite a byte to disk output buffer, writing the record to disk if buffer is full
@WRREC   B: Setzt DMA Puffer vor Schreiben eines RekordsSet DMA buffer before writing a record
  WRBUF   16 Bit DMA Puffer Adresse16 bit DMA buffer address
  WRBFP   8 Bit relative Pufferzeiger8 bit relative buffer pointer
  WRFCB   16 Bit FCB Adresse16 bit FCB address
  I: Die drei Felder müssen vor Aufruf gesetzt werden, falls nicht die Voreinstellungen verwendet werden sollen. Der Akku hält das Byte, das geschrieben werden sollThe three fields must be set up prior to calling the module, unless the defaults are to be used. The accumulator holds the byte to be written
  O: Die Carryflag ist gesetzt, wenn kein Platz mehr auf der Diskette vorhanden ist oder die Datei nicht gefunden wurdeCarry flag is set if there is no more space on the disk or the file was not found
GETEOF.MAC
GETEOF B: Lesen eines Zeichens von Diskette, testen des DateiendesRead a character from disk, testing for end of file
  I: ---
  O: Die Carryflag zeigt das Ende der Datei an, entweder physikalisch oder beim Zeichen EOF (Hex 1A - Ctrl-Z). Der Akku hält gelesenes ZeichenCarry flag indicates end of file, either physical or at the EOF character (Hex 1A - Ctrl-Z). The accumulator holds the character read
PUTEOF.MAC
PUTEOF B: Schreiben des Endes einer DateiWrite end of file and output current record
  I: ---
  O: Die Carryflag ist gesetzt falls kein Platz mehr auf Diskette ist oder die Datei nicht gefunden wurde, sonst nicht gesetzt. Der letzte Rekord der Datei wird auf Diskette geschrieben (Rest mit Ctrl-Z aufgefüllt). Ist in Adresse PUTEOF-1 das Bit 0 gesetzt, so wird zuvor noch ein EOF (Hex 1A - Ctrl-Z) geschrieben.Carry flag is set if there is no more space on the disk or the file was not found, otherwise reset. The last record of the file is written to disk (remainder filled with Ctrl-Z). If bit 0 at address PUTEOF-1 is set, an EOF (Hex 1A - Ctrl-Z) is written beforehand.

DSKUGT B: Rückschreiben eines gelesenen ZeichensUnread a byte from disk input buffer
  I: ---
  O: Die Carryflag zeigt Fehler an.Carry flag indicates error.
CLOSEF.MAC
CLOSEF B: Beenden und Schließen einer geöffneten Ausgabe DateiWrite end of file and close an open output file
  I: ---
  O: Carryflag gesetzt falls Datei nicht geschlossen werden konnte, sonst nicht gesetzt.Carry flag set if the file could not be closed, otherwise reset.
