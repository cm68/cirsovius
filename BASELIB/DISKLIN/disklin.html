Routinen für zeilenorientierte DiskettenoperationenRoutines for line-oriented disk operations

Die hier vorgestellten Routinen sind solche für zur zeilenweisen Ein-/Ausgaberoutinen mit Zeichen:The routines presented here are for line-oriented input/output operations with characters:
Modul-EntryModule Entry B: Beschreibung,Description, I: Eingabe-,Input, O: AusgabeparameterOutput parameters
FILLIN.MAC
FILLIN FILMOD B: Einlesen einer Zeile von Diskette bis Line FeedGet line from disk up to line feed
  I: Registerpaar DE hält Adresse auf den Zeilenpuffer, wobei das erste Byte in dem Puffer die maximale Anzahl einzulesender Zeichen angibt. Register B hält Zeichen, das an das Ende des Puffers gespeichert wird (üblicherweise „$" oder Null). Die Dateieingaberoutine kann geändert werden durch neue Adresse in FILMOD (Voreinstellung: GETEOF).Register pair DE points to the input buffer, where the first byte in the buffer holds the maximum number of characters to be read. Register B holds the character which will be stored at the end of the buffer (usually „$" or zero). The file input routine can be changed by placing a new address in FILMOD (default: GETEOF).
  O: Die Prozessorflags sind wie folgt gesetzt:
Zeroflag ist gesetzt bei leerem Puffer. Carryflag ist gesetzt bei Ende der Datei aber eingelesener Zeile. (Carry und Zero gesetzt zeigen dann das komplette Dateiende an). Der Akku hält die Länge der Zeile inklusive Carriage Return und Line Feed. Register C hält das zuletzt gelesene Zeichen
The processor flags are set as follows:
Zero flag is set on an empty buffer. Carry flag is set on end of file found but with a line already read. (Carry and zero both set indicate the complete end of file). The accumulator holds the character count including carriage return and line feed. Register C holds the last character read
EMPLIN.MAC
EMPLIN EMPMOD B: Ausgabe einer Zeile auf DiskettePut line to disk
  I: Registerpaar DE zeigt hält Adresse auf den Zeilenpuffer Register B hält Zeichen, das das Ende des Puffers anzeigt (üblicherweise „$" oder Null). Die Dateieausgaberoutine kann geändert werden durch neue Adresse in EMPMOD (Voreinstellung: DSKPUT).Register pair DE points to the input buffer. Register B holds the character which indicates the end of the buffer (usually „$" or zero). The file output routine can be changed by placing a new address in EMPMOD (default: DSKPUT).
  O: Im Fehlerfall ist die Carryflag gesetzt, sonst nicht.On error the carry flag is set, otherwise it is reset.
GETLNF.MAC
GETLNF B: Einlesen einer Zeile von Diskette bis Line FeedGet line from disk up to line feed
  I: Registerpaar DE zeigt hält Adresse auf den Zeilenpuffer, wobei das erste Byte in dem Puffer die maximale Anzahl einzulesender Zeichen angibt. Register B hält Zeichen, das an das Ende des Puffers gespeichert wird (üblicherweise „$" oder Null).
Die Dateieeingaberoutine muss vor dem Aufruf bestimmt werden durch neue Adresse in GETLNF-2Register pair DE points to the input buffer, where the first byte in the buffer holds the maximum number of characters to be read. Register B holds the character which will be stored at the end of the buffer (usually „$" or zero).
The file input routine must be specified before the call by placing a new address in GETLNF-2
  O: Die Prozessorflags sind wie folgt gesetzt:
Zeroflag ist gesetzt bei leerem Puffer. Carryflag ist gesetzt bei Ende der Datei aber eingelesener Zeile. (Carry und Zero gesetzt zeigen dann das komplette Dateiende an). Der Akku hält die Länge der Zeile inklusive Carriage Return und Line Feed, ebenso das zweite Byte des Puffers. Register C hält das zuletzt gelesene Zeichen
The processor flags are set as follows:
Zero flag is set on an empty buffer. Carry flag is set on end of file found but with a line already read. (Carry and zero both set indicate the complete end of file). The accumulator holds the character count including carriage return and line feed, as does the second byte of the buffer. Register C holds the last character read
