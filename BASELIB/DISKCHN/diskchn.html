Routinen für kanalorientierte DiskettenoperationenRoutines for channel oriented disk operations

Die hier vorgestellten Routinen sind solche für zur Zeichen Ein-/Ausgabe bei Diskettenoperationen. Die Auswahl der Dateien erfolgt hier über logische Kanäle, so wie es auch von BASIC bekannt ist:The routines presented here are those for character input/output in disk operations. The selection of files is done via logical channels, as is also known from BASIC:
Modul-EntryModule Entry B: BeschreibungDescription, I: Eingabe-Input, O: AusgabeparameterOutput parameters
CHNINI.MAC
CHNINI B: Initialisierung eines DiskettenkanalsInitialize a disk channel
  I: Register C hält Kanal in Bereich 0 bis 15
Register B hält Mode Byte wie folgt:
0 :Öffnen einer Datei zum Lesen
1 :Öffnen einer neuen Datei
2 :Datei löschen vor neuer Öffnung
3 :Öffnen einer Datei zum Schreiben
Bei gesetztem MSB kann eine Re-ininitialisierung eines bereits geöffneten Kanals erfolgen. Registerpaar DE hält Parameter Block Adresse:
PB+0,1die 16 Bit DMA Puffer Adresse
PB+2der 8 Bit relative Puffer Zeiger
PB+3,4die 16 Bit FCB Adresse
PB+5der Userbereich (-1 eingestellter)
Register C holds channel number in range 0 to 15
Register B holds mode byte as follows:
0 :Open an existing file for read
1 :Create a new file for write
2 :Delete file before creating new one
3 :Open an existing file for write (append)
If MSB is set, a re-initialization of an already opened channel can be performed. Register pair DE holds the parameter block address:
PB+0,1the 16 bit DMA buffer address
PB+2the 8 bit relative buffer pointer
PB+3,4the 16 bit FCB address
PB+5the user area (-1 for current)
  O: Die Carryflag ist bei einem Fehler gesetzt - Kanal außerhalb des Bereiches oder bereits geöffnet, Datei ist nicht auf Diskette oder kann nicht neu eingerichtet werdenCarry flag is set on error - channel out of range or already opened, file does not exist on disk or cannot be newly created
CHNTAB.MAC
@CHACC B: Überprüfen eines DiskettenkanalsValidate a disk channel
  I: Register C hält Kanal im Bereich 0 bis 15Register C holds channel number in range 0 to 15
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches. Bei nichtgesetzter Carryflag zeigt die gesetzte Zeroflag an, dass der Kanal geöffnet ist, sonst nicht geöffnet. Registerpaar HL zeigt auf den zugehörigen Kanal Kontroll BlockCarry flag is set on error - channel out of range. On carry reset, zero flag set indicates the channel is opened, otherwise not opened. Register pair HL points to the associated channel control block
CHNXFR.MAC
CHNXFR B: Durchführen eines Kanaltransfers. Lesen oder Schreiben ergibt sich aus der InitialisierungPerform a channel transfer. Read or write is determined by the initialization
  I: Register C hält Kanal im Bereich 0 bis 15 und Akku hält Byte, falls auf Diskette geschrieben werden sollRegister C holds channel number in range 0 to 15 and Accu holds byte if writing to disk
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches oder Dateifehler. Bei nicht gesetzter Carryflag hält der Akku das Byte, falls von Diskette gelesen wurdeCarry flag is set on error - channel out of range or file error. On carry reset, Accu holds the byte if read from disk
CHNIN.MAC
CHNIN B: Lesen eines Bytes vom DiskettenkanalRead a byte from disk channel
  I: Register C hält Kanal im Bereich 0 bis 15Register C holds channel number in range 0 to 15
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches oder Dateifehler, sonst nicht gesetzt und der Akku hält das gelesene ByteCarry flag is set on error - channel out of range or file error, otherwise not set and Accu holds the byte read
CHNOUT.MAC
CHNOUT B: Schreiben eines Bytes auf DiskettenkanalWrite a byte to disk channel
  I: Register C hält Kanal im Bereich 0 bis 15 und der Akku hält das zu schreibende ByteRegister C holds channel number in range 0 to 15 and Accu holds the byte to be written
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches oder DateifehlerCarry flag is set on error - channel out of range or file error
CHNEOF.MAC
CHNEOF B: Lesen eines Zeichens vom Diskettenkanal und testen EOFRead a character from disk channel and test for EOF
  I: Register C hält Kanal im Bereich 0 bis 15Register C holds channel number in range 0 to 15
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches oder Dateifehler - oder beim Zeichen EOF (Hex 1A - Ctrl-Z). Der Akku hält das gelesene ZeichenCarry flag is set on error - channel out of range or file error - or on EOF character (Hex 1A - Ctrl-Z). Accu holds the character read
CHNCLS.MAC
CHNCLS B: Schließen eines oder aller DiskettenkanäleClose one or all disk channels
  I: Register C hält Kanal im Bereich 0 bis 15 oder -1 falls alle Kanäle geschlossen werden sollenRegister C holds channel number in range 0 to 15 or -1 if all channels should be closed
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches oder DateifehlerCarry flag is set on error - channel out of range or file error
CHNUSR.MAC
@CHUSR B: Auswählen eines Userbereiches und Diskettenoperation ausführenSelect a user area and perform disk operation
  I: Registerpaar BC hält die Adresse der auszuführenden Diskettenoperation.
Registerpaar HL hält die Definitionsadresse-1 des Userbereiches. Userbereich -1 bedeutet keine Änderung des BereichesRegister pair BC holds the address of the disk operation to be performed.
Register pair HL holds the definition address-1 of the user area. User area -1 means no change of area
  O: Die Carryflag ist gesetzt im FehlerfallCarry flag is set on error
CHNTST.MAC
@CHTST B: Test ob der aufgerufene Kanal geöffnet istTest if the requested channel is opened
  I: Register C hält Kanal im Bereich 0 bis 15Register C holds channel number in range 0 to 15
  O: Die Carryflag ist gesetzt im Fehlerfall - Kanal außerhalb des Bereiches oder nicht geöffnet. Bei nicht gesetzter Carryflag zeigt die gesetzte Zeroflag einen Schreibkanal, eine nicht gesetzte Zeroflag einen Lesekanal anCarry flag is set on error - channel out of range or not opened. On carry reset, zero flag set indicates a write channel, zero flag not set indicates a read channel
