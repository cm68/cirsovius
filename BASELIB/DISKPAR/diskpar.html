Routinen für zeichenweise DiskettenoperationenRoutines for character-based disk operations

Die hier vorgestellten Routinen sind solche für zur Zeichen Ein-/Ausgabe bei Diskettenoperationen. Die BDOS-Schnittstelle ist in einem sechs Byte großen Parameterblock abgelegt (Siehe dazu die Module UDSKGT und UDSKPT). Gegenüber den vorigen Routinen erlauben diese die flexiblere Übergabe des Parameterblocks weil nun nur noch eine Adresse (zwei Bytes) getauscht werden muss. Außerdem wird die User Area unterstützt.The routines presented here are for character input/output during disk operations. The BDOS interface is stored in a six-byte parameter block (see modules UDSKGT and UDSKPT). Compared to the previous routines, these allow more flexible passing of the parameter block because now only one address (two bytes) needs to be exchanged. In addition, the user area is supported.

  RDPBLK   Zeiger auf einen sechs Byte Parameter Block:
PB+0,1die 16 Bit DMA Puffer Adresse

Pointer to a six-byte parameter block:
PB+0,1the 16-bit DMA buffer address
PB+2the 8-bit relative buffer pointer
PB+3,4the 16-bit FCB address
PB+5the user area (-1 means current)
  I: Der Parameterblock muss gesetzt sein, falls nicht die Defaultwerte verwendet werden sollenThe parameter block must be set up if the default values are not to be used
  O: Die Carryflag ist gesetzt falls Datei nicht vorhanden ist oder keine Rekords mehr existieren, sonst nicht gesetzt und Akku hält gelesenes ZeichenThe carry flag is set if the file is not found or no more records exist, otherwise reset and the accumulator holds the byte read

  I: ---
  O: Die Carryflag ist gesetzt bei Ende der Datei, physikalisch oder beim Zeichen EOF (Hex 1A - Ctrl-Z)The carry flag is set at end of file, either physical or on the character EOF (hex 1A - Ctrl-Z)
UPTEOF.MAC
UPTEOF B: Beenden einer Dateiausgabe auf beliebigem UserbereichFinalize a file output on any user area
  I: ---
  O: Die Carryflag ist gesetzt falls kein Platz mehr auf Diskette ist oder Datei nicht gefunden wurde, sonst nicht gesetzt. Der letzte Rekord wird mit Zeichen EOF gefüllt und auf Diskette geschrieben. Ist in Adresse UPTEOF-1 das Bit 0 gesetzt, so wird zuvor noch ein EOF (Hex 1A - Ctrl-Z) geschrieben.The carry flag is set if there is no more space on the disk or the file was not found, otherwise reset. The last record is filled with EOF characters and written to disk. If bit 0 at address UPTEOF-1 is set, an EOF (hex 1A - Ctrl-Z) is written beforehand.
