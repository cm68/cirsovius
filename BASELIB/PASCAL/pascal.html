Die hier vorgestellten Routinen sind solche für die Verabeitung von Routinen, die an die Sprache Pascal angelehnt sind:The routines presented here are those for processing routines modeled after the Pascal language:

POS.MAC
@POS B: Suche nach Teilkette einer ZeichenketteFind position of substring within target string
  I: Registerpaar HL zeigt auf Originalkette und Registerpaar DE auf Teilkette.Register pair HL points to the target string and register pair DE points to the substring.
  O: Register B hält die Position beginnend mit 1 und die Zeroflag ist gesetzt. Andernfalls hält B Null. Register HL zeigt auf die Teilkette innerhalb der Zeichenkette.Register B holds the position starting at 1 and the zero flag is set. Otherwise B holds zero. Register HL points to the substring within the target string.
COPY.MAC
@COPY B: Teilkette ausgebenCopy part of one string to another
  I: Registerpaar HL hält Originalkette, Registerpaar DE hält Teilkette, Register B hält zu kopierende Anzahl und Register C hält die Startposition in der Originalkette.Register pair HL holds the target string, register pair DE holds the destination string, register B holds the number of characters to copy, and register C holds the start position within the target string.
  O: Die gesetzte Carryflag zeigt einen Fehler der Startposition an, d.h. Position ist größer als Länge der Kette.A set carry flag indicates a start position error, i.e. position is greater than the length of the string.

  I: Registerpaar HL zeigt auf Originalkette und Registerpaar DE zeigt auf neue KetteRegister pair HL points to the source string and register pair DE points to the destination string.
  O: Originalzeichenkette in neue Kette kopiert. Die Registerpaare DE und HL weisen jeweils auf die abschließende NullSource string copied to destination string. Register pairs DE and HL each point to the terminating zero.

Die Registerpaare DE und HL weisen jeweils auf die abschließende Null, HL aber nur, wenn Kette komplett kopiert wurde.Source string copied to destination string. ASSIGN.MAC
@ASSIG B: Zuweisung einer DateiAssign a file name
  I: Registerpaar DE zeigt auf File Control Block mit 36 Bytes und Registerpaar HL zeigt auf Zeichenkette mit Dateinamen, Extension und optionalem Laufwerk.Register pair DE points to a File Control Block of 36 bytes, and register pair HL points to a string containing the file name, extension, and optional drive.
  O: Die gesetzte Carryflag zeigt einen Fehler in der Zeichenkette an.A set carry flag indicates an error in the string.
PARCNT.MAC
@PARCN B: Angabe der Parameter bei ProgrammaufrufCount parameters from program invocation
  I: Der Standard CP/M Buffer in Adresse 0080H hält eine Zeichenkette bestehend aus einer Anzahl durch Leerzeichen getrennter Parameter. Der Standard Buffer kann geändert werden durch neue Adresse in @PARCN-2The default CP/M buffer at address 0080H holds a string consisting of a number of parameters separated by spaces. The default buffer can be changed by storing a new address at @PARCN-2.
  O: Das Register B und der Akku halten die Anzahl der gefundenen Parameter. Die Zeroflag zeigt an, dass kein Parameter gefunden wurdeRegister B and the accumulator hold the number of parameters found. The zero flag indicates that no parameter was found.
PARSTR.MAC
@PARST B: Ausgabe eines Parameters aus Standard BufferSelect a parameter from the default buffer
  I: Der Standard CP/M Buffer in Adresse 0080H hält eine Zeichenkette bestehend aus einer Anzahl durch Leerzeichen getrennter Parameter. Der Standard Buffer kann geändert werden durch neue Adresse in @PARCN-2. Das Registerpaar DE hält Parameterkette und Register B hält die Nummer des gesuchten ParametersThe default CP/M buffer at address 0080H holds a string consisting of a number of parameters separated by spaces. The default buffer can be changed by storing a new address at @PARCN-2. Register pair DE holds the parameter string, and register B holds the number of the requested parameter.
  O: Die Parameterkette ist mit gesuchtem Parameter gefüllt oder leer falls die Nummer größer als die Anzahl Parameter ist.The parameter string is filled with the requested parameter, or empty if the number is greater than the total number of parameters.
PARSTRX.MAC
@PARSX B: Ausgabe eines Parameters aus Standard BufferSelect a parameter from the default buffer
  I: Der Standard CP/M Buffer in Adresse 0080H hält eine Zeichenkette bestehend aus einer Anzahl durch Leerzeichen getrennter Parameter. Der Standard Buffer kann geändert werden durch neue Adresse in @PARCN-2. Das Registerpaar DE hält Parameterkette, Register B hält die Nummer des gesuchten Parameters und Register C die Anzahl maximal zu lesender Zeichen.The default CP/M buffer at address 0080H holds a string consisting of a number of parameters separated by spaces. The default buffer can be changed by storing a new address at @PARCN-2. Register pair DE holds the parameter string, register B holds the number of the requested parameter, and register C holds the maximum number of characters to read.
  O: Die Parameterkette ist mit gesuchtem Parameter gefüllt oder leer falls die Nummer größer als die Anzahl Parameter ist. Die Zeroflag ist gesetzt, wenn alle Zeichen gelesen wurden, andernfalls hält das Register C die Anzahl nicht gelesener Zeichen.The parameter string is filled with the requested parameter, or empty if the number is greater than the total number of parameters. The zero flag is set when all characters have been read; otherwise register C holds the number of unread characters.
